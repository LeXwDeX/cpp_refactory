# C++ 开发硬约束 — cpp-dev-constraints

适用于所有 C++ 代码变更（重构、新功能、Bug 修复）。这些约束不可违反、不可跳过、不可部分执行。

---

## 版本兼容性

| 约束 | 说明 |
|------|------|
| `CPP_STANDARD` 是天花板 | 由 `cpp-scan.sh` 检测，所有代码变更不得使用该标准不支持的特性 |
| 不兼容必须标注降级方案 | 如需 C++17 特性但项目是 C++11，必须提供 C++11 等效实现 |
| 禁止"先用新特性后续再改" | 版本不兼容的代码不允许进入代码库 |

### 版本特性速查

| 特性 | 最低标准 | 降级方案 |
|------|----------|----------|
| `auto` | C++11 | 显式类型声明 |
| `nullptr` | C++11 | `0` 或 `NULL`（标注风险） |
| `std::unique_ptr` | C++11 | RAII wrapper 类 |
| `std::shared_ptr` | C++11 | 引用计数 wrapper（需证明必要性） |
| `constexpr` | C++11（扩展 C++14） | `const` + 运行时计算 |
| `std::optional` | C++17 | 哨兵值 + bool 标志 |
| `std::string_view` | C++17 | `const std::string&` |
| `if constexpr` | C++17 | SFINAE / 模板特化 |
| `std::span` | C++20 | 指针 + 长度参数 |

---

## 所有权规则

| 约束 | 说明 |
|------|------|
| 禁止引入 `std::shared_ptr` | 除非原代码本身就是引用计数的 |
| 优先 `std::unique_ptr` | C++11+ 项目的默认智能指针选择 |
| C++03 用 RAII wrapper | 封装裸指针，析构函数负责释放 |
| 每个 `new` 必须有明确所有者 | 不允许"谁最后用谁释放"的模糊所有权 |
| 禁止裸 `delete` | 必须通过智能指针或 RAII 析构 |

---

## 性能约束

| 约束 | 说明 |
|------|------|
| 禁止不必要的虚函数调用 | V-Table 消耗在热点路径不可接受 |
| 禁止不必要的堆分配 | 栈对象优先，堆分配需证明必要性 |
| 热点路径零额外开销 | 重构后的热点路径性能不得低于变更前 |
| 禁止不必要的拷贝 | 大对象传 `const&` 或 `&&`（C++11+） |

---

## 宏处理

| 约束 | 说明 |
|------|------|
| 保留所有条件编译逻辑 | `#ifdef` 路径不得删除 |
| 宏可抽象但保留编译期分支 | 可以用策略模式替代运行时宏，但 `#ifdef` 条件编译必须保留 |
| 不理解宏展开不删除宏 | 必须先用 `clang -E` 或 `clang_ast_macro_jungle` 确认展开结果 |
| 功能性宏标注风险 | 带副作用的宏标注 `// MACRO-SIDE-EFFECT:` |

---

## 安全性

| 约束 | 说明 |
|------|------|
| 检查 UB | 迭代器失效、悬空引用、use-after-free |
| `const_cast` / `reinterpret_cast` 必须标注 | 附带 `// why:` 注释说明理由和安全性证据 |
| off-by-one 不靠静态分析 | clang-tidy / cppcheck / scan-build 均无法检出，必须靠 ASan + 特征化测试 |
| 数组越界必须有边界检查 | 不使用裸数组，或确保所有访问有边界检查 |

---

## 代码风格

| 约束 | 说明 |
|------|------|
| 匹配所在文件的现有风格 | 缩进、命名、注释风格与周围代码一致 |
| 不引入新的命名约定 | 除非项目有明确的风格迁移计划 |
| 头文件 include 顺序保持一致 | 按项目现有顺序（通常是：自身头文件 → 项目头文件 → 系统头文件） |

---

## 错误处理

| 约束 | 说明 |
|------|------|
| 匹配现有错误处理模式 | 项目用返回值就不引入异常，用异常就不改为返回值 |
| 错误码必须有文档 | 新增错误码必须在头文件或文档中说明含义 |
| 不吞错误 | 捕获异常后必须处理（日志/重抛/转换），不允许空 catch |

---

## 测试约束

| 约束 | 说明 |
|------|------|
| gtest 必须 C++14 编译 | 即使被测代码是 C++03，测试文件必须用 C++14+ |
| 特征化测试先于重构 | 锁定现有行为，重构后验证行为未变 |
| 新功能必须有测试 | 无测试的新功能不交付 |
| Bug 修复必须有回归测试 | 锁定修复，防止复发 |

---

## 工具链约束

| 工具 | 已知限制 |
|------|----------|
| off-by-one 检测 | clang-tidy / cppcheck / scan-build 均无法检出，需 ASan |
| IWYU 跨版本 | IWYU 0.21 基于 clang-17，系统 clang-18 时需注意兼容性 |
| ast-grep 别名 | 必须用 `ast-grep` 全名，`sg` 被 screen group 占用 |

---

## 记忆写入

发现项目特有的约束时，写入持久记忆：

```
subject = "{项目名}"
predicate = "constraint"
object = "{约束描述}"
```

示例：
```
subject = "legacy-game-engine"
predicate = "constraint"
object = "所有渲染函数必须在主线程调用，不可从 worker 线程调用"
```
